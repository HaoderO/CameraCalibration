# 《相机标定零基础入门教程》插图重绘设计

## 1. 目标

将 `相机标定零基础入门教程.md` 中现有 ASCII 示意图、公式流程、算法流程和硬件流水线统一重绘为适合零基础读者的现代技术信息图，并以独立 SVG 文件嵌入 Markdown。

重绘必须同时满足：

- 不改变教程的技术结论；
- 降低坐标变换、畸变、优化、双目几何与硬件流水线的理解门槛；
- 保持整本教程的视觉语言一致；
- 支持无损缩放、中文检索和后续人工修改；
- 即使图片无法加载，正文仍保留必要的公式和流程信息。

## 2. 视觉方向

采用“白底黑线的现代技术信息图”风格：

- 纯白色背景；
- 黑色或近黑色表示主结构、坐标轴、轮廓、文字和主要数据流；
- 蓝色表示投影关系、辅助线或可选路径；
- 橙色表示关键点、观测值和当前步骤；
- 绿色表示正确、完成或理想状态；
- 红色表示误差、畸变和风险；
- 浅灰色用于网格、次要边框和背景分区；
- 使用规则几何、圆角信息框和统一箭头；
- 中文标签优先，保留通用数学符号与变量名；
- 彩色只承担语义强调，不承担主体结构，确保灰度打印时仍能完整辨读；
- 避免深色背景、霓虹效果和装饰性渐变，兼顾屏幕阅读与打印。

每张图包含图号、标题和一句图注。图内文字以简短标签为主，不复制正文段落。

## 3. 文件与集成规范

- 插图目录：`assets/camera-calibration/`
- 文件格式：独立 SVG
- 文件名：两位序号加英文语义名，例如 `01-pinhole-camera.svg`
- 常规画布：`1200 × 700`
- 复杂流程图允许增加高度，但宽度保持 1200
- SVG 必须设置 `viewBox`，不得依赖外部字体、脚本、位图或网络资源
- Markdown 使用相对路径引用，并提供准确的中文替代文本
- 图片下方添加简短斜体图注
- 现有 ASCII 图由 SVG 替换
- 现有公式与伪代码正文保留，新增插图只承担解释和组织作用

## 5. 信息组织原则

- 一张图只回答一个核心问题。
- 复杂主题采用从左到右或从上到下的单向阅读顺序。
- 流程步骤使用连续编号，避免交叉箭头。
- 同一变量在全书中保持同一种颜色和写法。
- 坐标轴必须标清方向；相机坐标系采用常见的 `X` 向右、`Y` 向下、`Z` 向前。
- 失真图必须用规则网格作参照，避免仅靠外轮廓暗示。
- 双目图必须明确左相机、右相机、基线、视差和深度之间的关系。
- 硬件图区分像素数据、坐标数据、存储访问和控制参数。
- 对零基础读者可能陌生的符号，在图中提供就近解释，不只放图例。

## 6. Markdown 修改策略

插图放在首次完整解释对应概念的段落之后。若某个代码块本身承担视觉说明，则用 SVG 替换；若代码块包含可复制的公式、命令或伪代码，则保留代码块，并在其前后插入解释图。

不重写教程主体，不调整章节顺序，不扩展教程范围。只允许为衔接插图增加图注或一句引导语。

## 7. 验证标准

完成后执行以下检查：

1. 所有 SVG 均为有效 XML，并含 `viewBox`。
2. 所有 Markdown 图片路径存在，且无未引用的正式插图。
3. 图片替代文本能够独立说明图的主题。
4. 中文标签没有裁切、重叠或超出画布。
5. 关键箭头、坐标轴和步骤编号在缩小显示时仍清晰。
6. 随机抽查成像、畸变、双目和硬件四类插图的实际渲染效果。
7. 原有 ASCII 示意图全部移除，公式和可复制流程信息仍保留。

## 8. 边界

本次不生成照片级素材，不引用网络图片，不修改教程中的代码实现，不重排全文版式，也不生成 PDF 或 Word 版本。
