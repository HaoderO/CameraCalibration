# Doc Generator — 文档生成器

根据设计需求或已有设计数据，生成符合项目规范的各类文档。

## 输出格式

| 格式 | 适用场景 | 生成方式 |
|---|---|---|
| Markdown (.md) | 设计文档、README、寄存器描述 | `Write` 工具直接写入 |
| DrawIO (.drawio) | 架构框图、时序图、状态机图 | 生成 mxGraph XML 写入 .drawio 文件 |
| Excel (.xlsx/.csv) | 寄存器列表、pinout 表、参数表 | Markdown 表格或 CSV 格式 |
| Word (.docx) | 正式设计规格书 | 先生成 Markdown 再导出 |
| PDF | 交付文档 | 先导出 Markdown 再用 pandoc 转换 |

## 设计文档规范

所有生成的设计文档必须遵循以下规范：

- Markdown 编写，结构清晰，层级分明
- 寄存器表格格式与 TRM 对齐（位域/属性/复位值/描述）
- 架构图用 DrawIO 绘制，源文件随文档一起提交到 `doc/` 对应目录
- 文档中引用芯片参考手册 (TRM) 时标注章节号，引用算法时标注论文/文档链接
- 模块接口变更后同步更新相关设计文档
- **文档中避免出现参考方案的具体型号和公司名称，仅描述技术特征**

## 流程

### Step 1: 确定输出格式与内容

根据用户需求确定：
1. 文档类型（设计文档、寄存器表、架构图、接口说明）
2. 输出格式（Markdown / DrawIO / Excel / PDF）
3. 存放路径（遵循项目目录规范 `doc/spec/`、`doc/design/`、`doc/analysis/`）

### Step 2: 生成文档内容

#### Markdown 设计文档模板

```markdown
# 模块名 — 功能简述

## 功能概述

- 主要功能描述
- 关键特性列表

## 接口定义

| 端口名 | 方向 | 位宽 | 描述 |
|---|---|---|---|
| clk | input | 1 | 系统时钟 |
| rst_n | input | 1 | 异步复位，低有效 |

## 寄存器描述

| 寄存器名 | 偏移 | 位域 | 属性 | 复位值 | 描述 |
|---|---|---|---|---|---|
| CTRL | 0x00 | [0] | RW | 1'b0 | 使能控制 |

## 架构设计

（DrawIO 架构图引用）

## 时序说明

（关键时序参数和波形描述）
```

#### DrawIO 架构图

生成符合项目规范的 DrawIO XML 格式，包含：
- 模块框图（矩形 + 端口标注）
- 数据流向箭头
- 时钟域边界标注
- 模块实例名与 RTL 代码一致

#### 寄存器表格

| 寄存器名 | 偏移 | 位域 | 属性 | 复位值 | 描述 |
|---|---|---|---|---|---|
| CTRL | 0x00 | [0] | RW | 1'b0 | 使能控制 |
| CFG | 0x04 | [7:0] | RW | 8'h00 | 配置参数 |

#### 寄存器排布规范

生成寄存器描述文档时，必须遵循以下排布规则：

##### 1. 配对合并

逻辑上成对的参数合并到同一寄存器，减少地址空间，避免配置中间态：

| 成对关系 | 合并寄存器 | 位域布局 |
|----------|-----------|---------|
| 宽+高 | `XXX_SIZE` | [27:16]=h, [15:12] rsvd, [11:0]=w |
| X+Y 坐标 | `XXX_ORIGIN` | [27:16]=y, [15:12] rsvd, [11:0]=x |
| 行+列 | `XXX_GRID` | [14:8]=rows, [7] rsvd, [6:0]=cols |

布局风格项目内统一：尺寸类高半字 H、低半字 W；坐标类高半字 Y、低半字 X。

##### 2. 地址连续性

删除某个寄存器后，后续所有寄存器地址依次前移，保持紧凑排列，**不留空洞**。禁止将释放的地址标记为"保留（reserved）"后再在其后追加新寄存器——应直接占用空出的地址位。

修改前后对比：

| 操作 | 错误做法 | 正确做法 |
|------|---------|---------|
| 删除 0x04 的寄存器 | 0x00, ~~0x04~~, 0x08, 0x0C（留空洞） | 0x00, 0x04, 0x08（前移紧凑） |
| 新增寄存器 | 追加到末尾 0x10 | 优先填入空闲位置，或追加后重排 |

##### 3. 位域对齐

多 bit 配置项按 4/8/16 bit 边界排布，禁止跨边界字段：

- 1-bit：可放任意位置
- 2~4 bit：起始位为 4 的倍数（bit0/4/8/12/16/20/24/28）
- 5~8 bit：起始位为 8 的倍数（bit0/8/16/24）
- 9~16 bit：起始位为 16 的倍数（bit0/16）

间隙填 rsvd，读回 0。好处：软件可用 byte/halfword 掩码独立访问，仿真波形十六进制对齐。

##### 4. 命名风格

- 寄存器名：`大写前缀_功能`，如 `FRAME_SIZE`、`MESH_GRID`
- 位域信号：`cfg_功能_细节`，如 `cfg_frame_w`、`cfg_mesh_cols`
- 尺寸类后缀：`_w`/`_h`（宽高）、`_size`（合并寄存器）
- 坐标类后缀：`_x`/`_y`、`_origin`（合并寄存器）
- 网格类后缀：`_cols`/`_rows`、`_grid`（合并寄存器）

##### 5. 批注

寄存器描述中需包含以下辅助信息，帮助使用者正确理解和使用：

- **使用方法**：典型配置序列、读写顺序、常用值示例
- **使用约束**：字段间的依赖关系、对齐要求、有效范围、互斥条件
- **行业规则**：YUV 格式编码惯例（0=420/1=422/2=422I）、地址对齐惯例（base[1:0]=0）、帧尺寸 4-byte 对齐等行业通用约定

批注写在寄存器详细描述的表格之后，用加粗小标题区分。仅标注非常规信息，命名本身已表达的不再重复。

### Step 3: 写入文件并验证

1. 使用 `Write` 工具写入文件到对应目录
2. 确认文件格式正确、内容完整
3. 提醒用户检查关键信息准确性

## 文档目录映射

| 文档类型 | 存放路径 |
|---|---|
| 需求规格、算法分析 | `doc/spec/` |
| 架构设计、模块划分 | `doc/design/` |
| 寄存器分析、性能评估 | `doc/analysis/` |
| 综合报告 | `syn/reports/` |

```
为 FEC_CORE 模块生成设计文档，包含接口定义和寄存器表
```

**输出**：
```
已生成 doc/design/FEC_CORE_设计文档.md，包含：
- 模块功能概述
- 12 个端口定义
- 8 个寄存器描述
- 数据通路架构图引用
```
