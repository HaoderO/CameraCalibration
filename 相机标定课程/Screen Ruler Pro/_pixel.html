A pixel tale

Precision and calculations

The only "precise" metrix when it comes to computer screens is the so called "pixel". We know that there are, for example, 165 pixels horizontally and 1050 pixels vertically. When we know the horizontal and vertical size (as with the Screen Ruler itself) in pixels, we can say for sure that we will get precise readings in pixel. If the width of the ruler is 100 px and the height is 100 px, we know for sure that our ruler perimeter is 400 and the area is 10,000 square pixels. That's the best precision we can get from a display. But what happens if we rotate our ruler (as the inner ruler or any other rotated figure)? Well, then we know nothing. When we are measuring the length of a rotated line everything is calculated and rounded. There are no "diagonal" pixels. To get the length of a rotated line we calculate it with the pythagorean theorem, then we round the result (because there is no such thing as "half a pixel") and write the calculated and rounded number of pixels on screen. You get the idea. It is still precise at some level (because the pythagorean theorem doesn't lie) but one can never be sure what will happen after many such calculations.

Now, let's convert the length in pixels (even before the rounding) into inches. What is the number of pixels per inch? Is the pixel square? (Remember the calibration issue?) What is the rotation angle? Again: calculations. What precision we are looking for? How many digits after the decimal point? Going this way could lead to a headache(which was my status quo when I was writing the respective code for all those calculations).

To minimize the errors, I am performing all calculations in real numbers (any rounding is done just before writing values on screen). But I just cannot promise you the absolute precision. Noone can. There could be pixel more, or pixel less in some situations so, please, do not expect laser-instruments precision from this tool.

The forgotten pixel

There is another important note to be taken into account. The "zero" pixel. When you look at the Start Point menu (which controls the orientation of the tickers at top/left/bottom/right), you will see that you have 2 "strange" options: Ruler starts from 0 and Ruler starts from 1. By default the ruler starts with 0 (zero). In other words, the very first pixel of the ruler is pixel number 0. Although numbered as "0", it is real and existing pixel when it comes to the width of the ruler box. That may confuse you in some moment. If you don't see what I mean, do this:

1) Drag a guide from the left side of the ruler and position it right over the 400px line of the top/bottom ticker
2) click "Tools / Set Position" and set the width of the ruler to 400px
3) hide all tickers and hide the border too (see the View menu)

Do you see the problem? The guide disappeared. Click Shift+Right Arrow keys (that will make the ruler 1 pixel wider). The ticker says "200", The guide says "200" but the actual width of the ruler is 201.

If select "Ruler starts with 1" from the Start Point menu, there will be no zero-pixel on the tickers anymore, therefore the ticker lines and the ruler size should match. However, this could be strange. For that reason I am measuring only with figures (usually rectangle), knowing that each time it includes everything, including the left/right lines
