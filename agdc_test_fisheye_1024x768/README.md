## Mesh 表规格

### 数据格式 (36-bit per node)

```
┌──────────────────┬──────────────────┐
│   v[17:0]        │   u[17:0]        │
│   bits [35:18]   │   bits [17:0]    │
└──────────────────┴──────────────────┘
```

- **u12.5 signed**: 1 LSB = 1/32 px
- **SRAM 寻址**: `addr = iy × (mesh_cols+1) + ix`
- **加载文件**: `agdc_mesh_16x12_36bit_in_u64.bin`（**不是** `uv_fixed.bin`）

### 网格坐标范围

| 网格   | tile    | u 范围            | v 范围            | 节点          |
| ------ | ------- | ----------------- | ----------------- | ------------- |
| 16×12  | 40×40   | [346.8, 676.8]    | [193.3, 574.4]    | 17×13 = 221   |
| 32×24  | 20×20   | [346.8, 676.8]    | [193.3, 574.4]    | 33×25 = 825   |

> 所有节点均在鱼眼图像 [0, 1024]×[0, 768] 内，有效比例 100%。

## 硬件约束分析

```
单行 v_in 极差 max = 381 px >> line_buf_depth (典型值 70)
逐行 vspan (相邻行位移) = 0.17 px  ← 不是瓶颈
```

**结论: 离线模式必须。** 三个分辨率中行缓冲需求最大——处理单行输出需访问源图像中跨越 381 行的数据。

### Slice + ROI 降低行缓冲需求

| ROI 宽度    | 单行 v_in 极差 | line_buf≤70 时最大 slice_h | passes |
| ----------- | -------------- | -------------------------- | ------ |
| 640 (全宽)  | 381 px         | 不可行                     | —      |
| 320         | 190 px         | 不可行                     | —      |
| 160         | 89 px          | 32 行                      | 15     |
| 120         | ~70 px         | 40 行                      | 12     |

> 仅靠纵向 slice 无效（单行 v_in 极差不变）。必须同时缩小 **ROI 宽度** 才能降低行缓冲需求。

## 推荐寄存器配置

**离线模式, 16×12 tiles, NV12 格式:**

```
// ============================================================
// Step 1: 初始化
// ============================================================
WR AGDC_BASE + 0x04 = 0x00000000   // CTRL: enable_rectify=0

// ============================================================
// Step 2: 配置寄存器组
// ============================================================

// --- 帧尺寸 ---
WR AGDC_BASE + 0x08 = 0x03000400   // INPUT_SIZE:  h=768 (0x300), w=1024 (0x400)
WR AGDC_BASE + 0x0C = 0x00000000   // ROI_ORIGIN:  roi_x=0, roi_y=0
WR AGDC_BASE + 0x10 = 0x01E00280   // OUTPUT_SIZE: h=480 (0x1E0), w=640 (0x280)

// --- 网格 ---
WR AGDC_BASE + 0x14 = 0x00000C10   // MESH_GRID: mesh_rows=12, mesh_cols=16

// --- 格式 ---
WR AGDC_BASE + 0x18 = 0x00000000   // FORMAT_CTRL: NV12 in/out

// --- 总线地址 (16B 对齐) ---
WR AGDC_BASE + 0x1C = <src_y_addr>     // SRC_Y_BASE_ADDR
WR AGDC_BASE + 0x20 = <src_y_addr + 0x000C0000>  // SRC_UV_BASE_ADDR (Y + 786432)
WR AGDC_BASE + 0x24 = <dst_y_addr>     // DST_Y_BASE_ADDR
WR AGDC_BASE + 0x28 = <dst_y_addr + 0x0004B000>  // DST_UV_BASE_ADDR (Y + 307200)

// --- 行跨度 ---
WR AGDC_BASE + 0x2C = 0x00000400   // SRC_STRIDE: 1024 bytes/row
WR AGDC_BASE + 0x30 = 0x00000280   // DST_STRIDE: 640 bytes/row

// --- 中断 ---
WR AGDC_BASE + 0x34 = 0x00004646   // ISP_LINE_BUF_CFG: 离线模式默认
WR AGDC_BASE + 0x40 = 0x00000001   // INT_MSK: frame_done

// ============================================================
// Step 3: 启动
// ============================================================
WR AGDC_BASE + 0x04 = 0x00000005   // CTRL: enable_rectify=1 + start_frame=1

// Step 4: 等待 busy=0 或 frame_done 中断
```
